# RSS Bot для Telegram

Бот для автоматического парсинга RSS-лент и публикации постов в Telegram-канал.

## 🚀 Возможности

- **Автоматический парсинг RSS** с настраиваемым интервалом
- **Динамическое управление лентами** через команды бота
- **Защита от дубликатов** (SHA-256 хеширование)
- **Валидация URL** (защита от SSRF-атак)
- **Санитизация HTML-контента** (защита от XSS)
- **Оптимизированная работа с БД** (пул соединений, индексы)
- **Логирование событий**
- **Graceful shutdown**
- **Docker-контейнеризация** с best practices

## 📋 Требования

- Python 3.10+
- Telegram Bot Token
- Telegram Channel ID
- Admin User ID

## 🔧 Установка

### Вариант 1: Docker Compose (рекомендуется)

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd rss-telegram-bot
```

2. Создайте файл `.env`:
```bash
cp .env.example .env
```

3. Заполните `.env` своими данными:
```env
BOT_TOKEN=your_bot_token_here
CHANNEL_ID=-1001234567890
ADMIN_ID=123456789
CHECK_INTERVAL=300
DB_PATH=/app/data/seen_posts.db
MAX_DB_ENTRIES=10000
RSS_TIMEOUT=10
POSTS_TO_CHECK=5
```

4. Запустите бота:
```bash
docker-compose up -d --build
```

### Вариант 2: Локальный запуск

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Создайте и заполните `.env` (см. выше)

3. Запустите бота:
```bash
python bot.py
```

## 📖 Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Информация о боте и список команд |
| `/add <url>` | Добавить RSS-ленту |
| `/remove <url>` | Удалить RSS-ленту |
| `/feeds` | Список активных лент |
| `/stats` | Статистика бота |

## 🔒 Безопасность

### Реализованные меры защиты:

1. **SSRF Protection**: Валидация URL запрещает доступ к внутренним IP-адресам
2. **XSS Protection**: Санитизация HTML-контента из RSS
3. **Authentication**: Проверка ADMIN_ID для всех команд
4. **Input Validation**: Валидация всех входных данных
5. **SQL Injection Prevention**: Параметризованные запросы
6. **Token Sanitization**: Очистка токена от пробелов

## ⚡ Оптимизации производительности

1. **Пул соединений с БД**: Единое соединение переиспользуется
2. **WAL режим SQLite**: Улучшенная производительность записи
3. **Индексы**: Ускорение поиска по хешам
4. **Асинхронный парсинг**: Не блокирует event loop
5. **Таймауты**: Защита от зависаний при парсинге
6. **Очистка старых записей**: Автоматическое управление размером БД

## 🛠 Настройка

### Переменные окружения

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `BOT_TOKEN` | Токен Telegram бота | - |
| `CHANNEL_ID` | ID канала для публикаций | - |
| `ADMIN_ID` | ID администратора | - |
| `CHECK_INTERVAL` | Интервал проверки (сек) | `300` |
| `DB_PATH` | Путь к базе данных | `seen_posts.db` |
| `MAX_DB_ENTRIES` | Макс. записей в БД | `10000` |
| `RSS_TIMEOUT` | Таймаут парсинга (сек) | `10` |
| `POSTS_TO_CHECK` | Постов за проверку | `5` |

### Получение Channel ID

1. Добавьте бота в канал как администратора
2. Отправьте сообщение в канале
3. Перешлите сообщение боту @userinfobot или используйте @RawDataBot
4. ID будет в формате `-1001234567890`

## 🐛 Troubleshooting

### Бот не запускается
- Проверьте правильность `BOT_TOKEN` в `.env`
- Убедитесь, что все переменные окружения установлены

### Бот не публикует посты
- Проверьте, что бот добавлен в канал как администратор
- Убедитесь, что `CHANNEL_ID` указан правильно (с минусом)

### Ошибки парсинга RSS
- Проверьте доступность RSS-ленты
- Увеличьте `RSS_TIMEOUT` если лента медленно отвечает
